# Template: Maestro Mobile Test Flow

Source: generated by `/e2e-mobile` skill
Pattern: YAML-based mobile E2E test flow

```yaml
# .maestro/login-flow.yaml
appId: {bundle-id}
tags:
  - smoke
  - auth
---
# Login Flow — verifies full authentication journey

- launchApp:
    clearState: true

# Wait for app to load
- assertVisible: "Welcome"

# Navigate to login
- tapOn: "Sign In"
- assertVisible: "Login"

# Enter credentials
- tapOn: "Email"
- inputText: "test@example.com"
- tapOn: "Password"
- inputText: "Test1234!"
- hideKeyboard

# Submit
- tapOn: "Sign In"

# Verify success
- assertVisible: "Dashboard"
- takeScreenshot: screenshots/login-success

# Verify user data loaded
- assertVisible: "test@example.com"

# Test logout
- tapOn: "Profile"
- tapOn: "Sign Out"
- assertVisible: "Welcome"
- takeScreenshot: screenshots/logout-success
```

## Usage
Use when setting up mobile E2E testing via `/e2e-mobile`.

## Customization Points
- `{bundle-id}`: App bundle identifier (e.g., com.example.app)
- Screen names and button labels
- Test data (emails, passwords)
- Screenshot paths
