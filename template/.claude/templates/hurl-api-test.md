# Template: Hurl API Test Suite

Source: generated by `/api-test` skill
Pattern: HTTP request/response testing with assertions

```hurl
# tests/api/health.hurl
# Health check — should always return 200
GET {{base_url}}/health
HTTP 200
[Asserts]
jsonpath "$.status" == "ok"
duration < 200
```

```hurl
# tests/api/auth-register.hurl
# User registration flow
POST {{base_url}}/api/v1/auth/register
Content-Type: application/json

  "email": "test-{{timestamp}}@example.com",
  "password": "Test1234!",
  "name": "Test User"
}

[Asserts]
jsonpath "$.id" isString
jsonpath "$.email" contains "@"
duration < 1000

# CRUD operations on {resource}

  "name": "Updated Item"

```

## Usage
Use when setting up API testing via `/api-test`. Run with:
```bash
hurl --test --variable base_url=http://localhost:8000 tests/api/*.hurl
```

## Customization Points
- `{base_url}`: API base URL
- `{resource}`: REST resource name (e.g., items, users, orders)
- Auth mechanism (Bearer token, API key, cookie)
- Expected status codes and response shapes
